# OnBid ValueUp — 온비드 공매 적정가 산출기

> ⚠️ **비공식 개인 프로젝트** — 캠코(KAMCO)·온비드(OnBid) 공식 서비스와 **무관**합니다. 개인이 학습 목적으로 작성한 코드이며, 이 프로그램 사용으로 인한 어떠한 손해에 대해서도 제작자는 책임을 지지 않습니다. [DISCLAIMER](DISCLAIMER.md) 참조.

---

## 배경 및 목적

온비드(공매) 낙찰률이 낮은 이유 중 하나는 **정보 비대칭** 입니다.

담보신탁 물건의 경우 등기부 을구가 비어 있어도 **신탁원부에만 기재된 선순위 채무(우선수익금액)** 가 존재할 수 있습니다. 이 금액은 공매 공고문에도, 등기부등본에도 공개되지 않아 입찰자가 적정가를 계산할 수 없는 구조적 문제가 있습니다.

이 도구는 두 가지를 목표로 합니다:

1. **드러난 리스크 비용 자동 산출** — 명도 소송, 체납 관리비, 조세체납 특약 등 공개된 리스크를 수치화
2. **숨겨진 리스크 경고** — 신탁원부 미열람으로 우선수익금액이 불명확한 경우 계산을 차단하고 명시적으로 경고

## 고려하는 리스크 항목

| 항목 | 평균 비용 | 적용 조건 |
|------|-----------|-----------|
| 명도 소송 비용 | 300만원 | 임차인 존재 시 |
| 체납 관리비 인수 | 200만원 | 관리비 특약 시 |
| 위탁자 조세체납 인수 | 500만원 | 조세체납 특약 시 |
| 등기 불가 리스크 | 300만원 | 조세체납 특약 시 |
| 취득세 추가 부담 | 동적 계산 | 업무시설(4.6%) |
| 담보신탁 하자 할인 | 감정가 3% | 담보신탁 물건 |
| **신탁 우선수익금액 미확인** | **계산 차단** | **신탁원부 미열람** |

## 설치 및 실행

```bash
git clone https://github.com/<your-id>/kamco-onbid-valueup.git
cd kamco-onbid-valueup
python -m venv venv && source venv/bin/activate
pip install -r requirements.txt

# API 키 없이도 샘플 데이터로 실행 가능
python main.py --list          # 샘플 목록 확인
python main.py S-3068          # 담보신탁 오피스텔 (적정가 산출)
python main.py DEMO-001        # 캠코공매 리스크 없는 물건 (비교용)
python main.py MORAN-401       # ★ 신탁 우선수익금액 미공개 → 경고 출력
```

## 샘플 매물

### `S-3068` — 분당풍림아이원플러스오피스텔 (일반 분석)
- 담보신탁 물건, 리스크 비용 차감 후 적정가 산출
- 명도 소송, 관리비, 조세체납, 취득세 추가, 하자 할인 포함

### `DEMO-001` — 수원영통오피스텔 (리스크 없는 비교 케이스)
- 임차인 없음, 캠코공매, 특약 없음

### `MORAN-401` — 이안모란센트럴파크 102동 401호 ⚠ 경고 사례
- 경기도 성남시 중원구 | 담보신탁 (KB부동산신탁) | 6차 유찰
- 등기부 을구: **기록사항 없음** → 선순위 없어 보이지만 실제로는 신탁원부에 선순위 채무 존재
- 우선수익자: 부안수협·인천수협·죽변수협·포항수협·군산시수협 5개 기관
- 우선수익금액: **신탁원부에만 기재, 공고문·등기부에 미공개**
- 결과: 적정가 산출 **차단** + 신탁원부 열람 안내

## API 키 설정 (선택)

`.env.example`을 참고하여 `.env` 파일 생성:

```env
MOLIT_API_KEY=your_key_here    # 국토부 실거래가 API (data.go.kr)
ONBID_API_KEY=your_key_here    # 온비드 공매정보 API (data.go.kr)
```

API 키 없이도 내장 샘플 데이터로 모든 기능 작동합니다.

## 정책적 제언

이 프로젝트를 만들면서 발견한 구조적 문제:

- **신탁원부는 등기부등본과 별개 문서**로, 공매 공고에 의무 첨부 규정이 없음
- 입찰자가 우선수익금액을 알려면 등기소에서 신탁원부를 별도 열람해야 하며, 일부는 영구보존문서로 물리적 열람만 가능
- 결과적으로 **정보 비대칭 → 반복 유찰 → 공매 효율 저하**

온비드 공고에 신탁원부 핵심 정보(우선수익금액, 우선수익자)를 의무 공시하는 제도 개선이 필요합니다.
